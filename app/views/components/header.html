Header

Use the header component for Ofgem services that are not on GOV.UK branded.

When to use this component

  • For Ofgem applications and services
  • To provide consistent branding across Ofgem services
  • Extend this header to include a service name and navigation, and user information and account details

How it works

The header component provides consistent branding and navigation across Ofgem services. It displays the Ofgem logo and can include additional navigation elements.

When to use this component

  • Use on all Ofgem service pages to maintain brand consistency
  • When you need to provide clear service identification
  • For services that require Ofgem branding and logo display
  • When building internal or external Ofgem applications

When not to use this component

  • Don't use on non-Ofgem branded services
  • Avoid using on pages that don't require official Ofgem branding
  • Don't use if you need a different government department's branding

Component attributes

Attribute Type Required Description
classes string No Additional CSS classes to apply to the header. Use "ofgem-header" for Ofgem branding.
homepageUrl string Yes URL for the homepage link (usually "/").
containerClasses string No CSS classes for the header container. Default is "govuk-width-container".
attributes object No HTML attributes to apply to the header element (id, style, etc.).
logo object Yes Configuration for the logo display.
logo.href string Yes URL for the logo link (usually "/").
logo.text string Yes Text to display as fallback if image fails to load.
logo.image object Yes Image configuration for the logo.
logo.image.src string Yes URL of the logo image.
logo.image.alt string Yes Alt text for the logo image.
logo.image.width number No Width of the logo image in pixels.
logo.image.height number No Height of the logo image in pixels.

Customization options

Styling customization

  • Border removal: Use style: "border-bottom: none;" in attributes to remove the default bottom border
  • Custom classes: Add additional CSS classes via the classes parameter
  • Container width: Modify containerClasses to change the header container styling

Logo customization

  • Different logos: Change the logo.image.src to use different Ofgem logo variants
  • Logo sizing: Adjust width and height values to resize the logo
  • Alt text: Customize the alt text for better accessibility

Navigation customization

  • Homepage URL: Change homepageUrl to point to your service's homepage
  • Logo link: Modify logo.href to change where the logo links to

Accessibility considerations

  • Always provide meaningful alt text for the logo image
  • Ensure the logo text fallback is descriptive and matches the image
  • The header has proper semantic HTML with role="banner"
  • Logo links should have clear focus indicators
  • Maintain sufficient color contrast for any custom styling

Browser support

This component works in all modern browsers and follows GOV.UK Frontend browser support guidelines.

Header elements

  • Logo - Ofgem branding and link to homepage
  • Service name - Clear identification of the current service
  • Navigation - Main menu items for the service
  • User information - User name, role, and account links

Header types

  • Basic header - Logo and service name only
  • Navigation header - Includes main navigation menu
  • User header - Includes user information and account links
  • Branded header - Uses Ofgem logo and styling

Service naming

  • Use clear, descriptive service names
  • Keep service names short and specific
  • Use sentence case for service names
  • Avoid technical jargon in service names

Accessibility

  • Use the role="banner" attribute for the header element
  • Include proper alt text for logos and images
  • Ensure navigation is keyboard accessible
  • Use descriptive link text for navigation items
  • Provide skip links for keyboard users
  • Ensure sufficient color contrast for all text elements

Research

This component follows Ofgem Design System patterns and has been tested with users on LA Flex and GBIS. Headers are most effective when they provide clear navigation and consistent branding.
